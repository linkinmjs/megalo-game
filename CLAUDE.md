## Project Overview

**megalo-claude** is a Godot 4.6 game project. It is currently a blank/initial project with no scenes or scripts yet.

## Running the Project

This project has no custom build scripts. Everything runs through the Godot Editor:

- **Open project**: Launch Godot 4.6+ and open `project.godot`
- **Run from editor**: Press `F5` (run project) or `F6` (run current scene)
- **Run from CLI**: `godot --path . project.godot`
- **Export**: Use Godot's **Project > Export** menu to build for a target platform

There is no package manager, no `npm install`, no compile step — Godot handles everything internally.

## Technology Stack

- **Engine**: Godot 4.6
- **Language**: GDScript (primary), with optional C# or GDExtension support
- **Physics**: Jolt Physics (configured in `project.godot`)
- **Rendering**: GL Compatibility (cross-platform, includes mobile)
- **Windows driver**: D3D12

## Project Configuration (`project.godot`)

Key settings already configured:
- `config_version = 5` (Godot 4.x format)
- `rendering/renderer/rendering_method = "gl_compatibility"`
- Physics engine: Jolt

## Godot 4 Conventions

When adding code to this project, follow Godot 4 patterns:

- Scenes go in `res://scenes/` (create the folder if needed)
- Scripts go next to their scene or in `res://scripts/`
- Use `@export` for inspector-exposed variables
- Use signals with `signal my_signal(arg: Type)` syntax
- Prefer `@onready var node = $NodePath` over `get_node()`
- Autoloads (singletons) are declared in **Project Settings > Autoload**
- `.godot/` directory is auto-generated by the editor and should not be edited manually
